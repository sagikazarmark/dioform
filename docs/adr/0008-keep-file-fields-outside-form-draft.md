# Keep file fields outside the form draft

Dioform will model user-selected files as adapter-owned **File Selections** addressed by typed **File Field Keys**, rather than as `FieldPath<Model, File>` values inside the **Form Draft**. A **File Field** is field-like for metadata, validation errors, accessibility, submit availability, reset, and reinitialization, but selected files are not part of the **Form Model** and are exposed to application submit behavior through a file-aware submit-time snapshot. **File Field Keys** carry cardinality: single-file fields retain at most one selected file, while multi-file fields retain an ordered list. File-selection validators may be synchronous or asynchronous; async file-selection checks receive a submit-time file snapshot and participate in submit **PendingValidation** while still attaching errors to the file field identity. This keeps **Form Core** renderer-agnostic and file-handle-free, preserves cloneable form drafts and snapshots, avoids serializing platform file handles across hydration or form-state snapshots, and leaves native browser file POST interop as a separate boundary from **Dioxus-Managed Submission**. The trade-off is a dedicated file-selection API instead of reusing ordinary field paths, but that explicit seam matches the different ownership and lifecycle rules for platform files.
