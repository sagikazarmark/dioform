# Use browser submit preflight without async waiting

Dioform will model **Progressive Submission** as browser-owned POST with a hydrated **Browser Submit Preflight** that can block current known client-side blockers, rather than as a second managed submit lifecycle. The preflight runs synchronous submit validation and blocks mounted parse errors, existing submit-relevant pending validation, and in-flight submissions, but it does not start or wait for submit-only async validators; applications that need client async validation to finish before submit behavior runs should use **Dioxus-Managed Submission**. This preserves no-JS browser fallback and avoids inventing resubmit orchestration after async validation, while keeping the stricter typed lifecycle behind `managed_submit()`.
